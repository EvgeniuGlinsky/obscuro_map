# Handoff: Obscuro Map — UI Design

## Overview
Obscuro Map — мобильное приложение, в котором мир покрыт туманом войны из шестиугольных ячеек. Пользователь физически посещает места — и открывает их на карте. Цель этого пакета: передать визуальный дизайн трёх ключевых экранов разработчику для внедрения в нативное мобильное приложение.

## О файлах в пакете
Файл `Obscuro Map Design.html` — **дизайн-прототип в HTML**, созданный как визуальный референс. Это **не продакшн-код** — его не нужно копировать напрямую. Задача разработчика: воссоздать эти экраны в существующей кодовой базе приложения (Android/iOS/Flutter/React Native) с использованием её паттернов, библиотек и компонентов.

Как открыть прототип: просто откройте `Obscuro Map Design.html` в браузере. Можно масштабировать холст колёсиком мыши, кликать на лейблы артбордов для перехода в полноэкранный режим.

## Fidelity
**High-fidelity (hifi)** — точные цвета, типографика, отступы и поведение. Реализация должна воспроизводить дизайн пиксельно точно, используя существующие паттерны кодовой базы.

## Экраны / Views

### 1. Splash Screen (Заставка)

**Назначение:** Показывается при запуске приложения, пока грузится карта.

**Фон:**
- Тёмный градиент: `linear-gradient(168deg, #13101F 0%, #0D0B18 55%, #111626 100%)`
- Поверх фона — едва видимая сетка шестиугольников (flat-top, circumradius ~58px, stroke `#C1BDD2` opacity 5.8%)

**Центральный элемент — логотип (пин):**
- SVG: форма location pin (каплевидный силуэт), заполненная градиентом `linear-gradient(от верх-лево к низ-право, #9B8EC4 → #3D3360)`
- Внутри пина — сетка гексов `#C1BDD2` opacity 42%, обрезанная по форме пина (clipPath)
- Поверх — shine-слой: `radial-gradient(35% 28%, rgba(255,255,255,0.2) → transparent)`
- Внутренний круг: `rgba(11,8,20,0.58)`, радиус 16
- Маленькая точка внутри: `rgba(193,189,210,0.22)`, радиус 6
- Тень на всём пине: `drop-shadow(0 10px 36px rgba(107,90,155,0.42))`
- Итоговый размер: ~118×140pt

**Радиальное свечение за логотипом:**
- Круг 380×380pt, `radial-gradient(rgba(107,90,155,0.14) → transparent 70%)`
- Позиция: центр по X, смещение -60% по Y от центра экрана

**Текст:**
- `OBSCURO MAP` — шрифт **Cinzel**, 26pt, weight 600, letterSpacing 0.14em, цвет `#C1BDD2`
- `Explore the unknown` — шрифт **Cinzel**, 10pt, weight 400, letterSpacing 0.38em, цвет `rgba(193,189,210,0.38)`, uppercase
- Отступ между логотипом и названием: 34pt
- Отступ между названием и tagline: 10pt

### 2. Map Screen — Grid ON (Карта, сетка включена)

**Назначение:** Основной экран. Карта покрыта шестиугольным туманом войны. Сетка гексов видна.

**Фон карты (светлый, дневной стиль):**
| Элемент | Цвет |
|---|---|
| База (суша) | `#E8E3D8` |
| Водоём | `#AAD3DF` |
| Парки | `#C8DFAF` |
| Кварталы (чётные) | `#DADAD6` |
| Кварталы (нечётные) | `#D4CFC8` |
| Главные дороги | `#FFFFFF` (ширина 5px) |
| Второстепенные дороги | `#F0EDE6` (ширина 2.5px) |
| Малые дороги | `#EAE6DE` (ширина 1.2px) |

**Шестиугольная сетка (flat-top hexagons):**
- Circumradius: **R = 72pt**
- Column spacing (centre to centre): **R × 1.5 = 108pt**
- Row spacing (centre to centre): **R × √3 ≈ 124.7pt**
- Нечётные столбцы смещены вниз на **R × √3 / 2 ≈ 62.35pt**
- Рабочий радиус для рендера: **R − 2 = 70pt** (небольшой зазор между гексами)

**Туман войны (fog of war):**
- Неразведанные гексы: тёмный полупрозрачный слой `rgba(18,12,38,0.72)` поверх карты
- Туман рисуется через Canvas 2D `destination-out`: рисуем fog rect на весь экран, затем вырезаем разведанные гексы с помощью `globalCompositeOperation = 'destination-out'`
- **Важно:** туман полупрозрачный — карта под ним слабо видна

**Terrain Art (Civ-style иллюстрации в неразведанных гексах):**
Каждый неразведанный гекс содержит стилизованную геометрическую иллюстрацию в духе Civilization (только при включённой сетке). Цвет иллюстраций: `#C1BDD2`, opacity 38–42%, stroke 1.2–1.4pt, fill `rgba(193,189,210,0.05–0.08)`.

| Тип | Описание |
|---|---|
| `mountains` | 2 перекрывающихся треугольника (горы) + горизонтальная линия |
| `forest` | 3 ёлки (2 треугольника + ствол), расположены рядом |
| `water` | 3 горизонтальные волнистые линии + ellipse-контур |
| `ruins` | Прямоугольная башня с зубцами, арка, обломки по бокам |
| `plains` | Плавные кривые линии + точки |

**Линии сетки (при grid ON):**
- Разведанные гексы: stroke `rgba(193,189,210,0.28)`, strokeWidth 1pt
- Неразведанные гексы: stroke `rgba(193,189,210,0.15)`, strokeWidth 1pt

**Маркер пользователя:**
- Внешнее свечение: radialGradient `#9D8FCC`, radius 34pt, opacity 0→0.55
- Кольцо: radius 13pt, fill `rgba(107,90,155,0.22)`, stroke `rgba(193,189,210,0.35)` 1.5pt
- Внутренняя точка: radius 6pt, fill `#9D8FCC`
- Белый центр: radius 3pt, fill `#FFFFFF`

**Кнопка «Войти через Google»:**
- Позиция: top 66pt от верха экрана, horizontally centred
- Стиль: pill (borderRadius 9999), background `rgba(255,255,255,0.97)`
- Padding: 10pt вертикально, 20pt горизонтально
- Shadow: `0 4px 24px rgba(0,0,0,0.48), 0 1px 4px rgba(0,0,0,0.2)`
- Border: `1px solid rgba(255,255,255,0.45)`
- Иконка Google (SVG) + текст "Войти через Google", fontSize 15pt, fontWeight 500, цвет `#18142A`

### 3. Map Screen — Grid OFF (Карта, сетка скрыта)

**Идентична Grid ON**, но с двумя отличиями:

1. **Сетка скрыта** — линии гексов не рисуются, terrain art не показывается
2. **Разведанная зона** — одна цельная область без шестиугольных делений. При рендере на Canvas используйте `HR + 6 = 78pt` в качестве радиуса при вырезании гексов (вместо рабочего R−2=70pt), чтобы соседние гексы перекрывались и сливались в единую зону без видимых швов
3. **Кнопка сетки (шестая снизу)** — неактивна (белый фон), в отличие от Grid ON

## Панель управления (6 кнопок)

Расположение: правый край экрана, вертикально по центру, gap 9pt.

| # | Функция | Иконка |
|---|---|---|
| 1 | На моё местоположение | Перекрестие (crosshair) |
| 2 | Увеличить | Плюс |
| 3 | Уменьшить | Минус |
| 4 | Удалить разведанные точки | Корзина/мусорный бак |
| 5 | Закрасить замкнутые области | Ведро с краской (paint bucket) |
| 6 | Показать/скрыть сетку | Шестиугольник с внутренними линиями |

**Стиль кнопки (неактивная):**
- Размер: 48×48pt, borderRadius 24pt (круг)
- Background: `rgba(255,255,255,0.94)`
- Border: `1px solid rgba(255,255,255,0.55)`
- Shadow: `0 4px 20px rgba(0,0,0,0.44), 0 2px 8px rgba(0,0,0,0.28)`
- Иконка: цвет `#18142A`, strokeWidth ~1.9–2.2pt

**Стиль кнопки (активная — кнопка сетки при grid ON):**
- Background: `linear-gradient(145deg, #8B7FAA 0%, #5B4E8A 100%)`

## Логика рендера тумана войны

```
1. Создать canvas/surface размером с экран
2. Залить полностью: rgba(18,12,38,0.72)
3. Для каждого разведанного гекса:
   a. Grid ON:  вырезать гекс радиусом r=70pt (destination-out)
   b. Grid OFF: вырезать гекс радиусом r=78pt (destination-out)
      — увеличенный радиус создаёт перекрытие ~6pt между соседними гексами,
        их destination-out операции накладываются → единая бесшовная зона
4. Рисовать terrain art поверх (только при grid ON)
5. Рисовать линии сетки поверх (только при grid ON)
6. Рисовать маркер пользователя поверх всегда
```

## Примечания для разработчика
- Прототип создан на React + SVG + Canvas — не копируйте код напрямую, используйте его как визуальный референс
- Шестиугольная математика (hexPoints, hexCenter) в прототипе — рабочая и может быть портирована в нативный код
- Для бесшовного слияния разведанных гексов без сетки **обязательно используйте увеличенный радиус** при вырезании — это критично для UX
- Terrain art рекомендуется реализовать через векторные drawable/SVG assets, не через Canvas drawing
